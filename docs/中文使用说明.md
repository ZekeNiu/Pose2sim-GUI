# Pose2Sim GUI 中文使用说明

## 1. 是否需要 input 和 output 文件夹

需要建立，但它们属于本 GUI 的本地工作目录，不是 Pose2Sim 官方项目内部必须读取的目录。本 GUI 会在 `D:\Application\Biomechanics\Pose2sim` 下自动创建：

- `input/videos/`：原始录制视频暂存区，导入视频窗口默认从这里开始选择。
- `input/calibration/`：校准素材暂存区。
- `output/pose2sim_results/`：Pose2Sim 项目结果的集中位置。GUI 新建项目默认放在这里，内部包含 `pose/`、`pose-sync/`、`pose-associated/`、`pose-3d/`、`kinematics/`，以及 `reports/` 报告子目录。

这些目录已经被 `.gitignore` 忽略，不会推送到 GitHub。Pose2Sim 本身仍然读取每个项目内的 `videos/`、`calibration/` 和 `Config.toml`。普通用户不需要手写 `Config.toml`：在 GUI 中选择或新建一个项目文件夹后，如果里面没有 `Config.toml`，GUI 会自动复制 Pose2Sim 内置默认配置，并创建 `videos/`、`calibration/`、`reports/`。若项目是用 GUI 新建的，项目本身就在 `output/pose2sim_results/<项目名>/`，结果不会再额外复制一份；若你选择外部项目，GUI 会同步一份结果到 `output/pose2sim_results/<项目名>/`，但不会默认删除外部项目里的原始结果。

## 2. Pose2Sim 是否会输出 osim、mot 等文件

会。完成 OpenSim 运动学阶段后，Pose2Sim 会在项目的 `kinematics/` 文件夹中输出：

- `*_scaled.osim`：按被试尺度缩放后的 OpenSim 模型。
- `*.mot`：逆运动学得到的关节角/坐标时间序列。
- `opensim_logs.txt`：OpenSim 日志。

三维标记点轨迹通常输出到 `pose-3d/*.trc`。本 GUI 的 Excel 和 HTML 报告使用 `kinematics/*.mot` 作为数据源，并保存到同一结果项目下的 `reports/` 子目录。

## 3. 是否支持导入录制好的视频

支持。Pose2Sim 的常规流程就是处理提前录制好的多相机视频。使用本 GUI 时：

1. 打开或新建一个 Pose2Sim 项目。
2. 如果该文件夹没有 `Config.toml`，GUI 会自动创建默认配置和标准文件夹；已有配置则不会覆盖。
3. 点击“导入录制视频”，选择多个相机的视频文件。
4. 视频会被复制到项目的 `videos/` 文件夹。
5. 若项目是用 GUI 新建的，默认会放在软件目录 `output/pose2sim_results/` 下，便于集中查看结果和报告。

建议命名为 `cam01.mp4`、`cam02.mp4`、`cam03.mp4`，便于和校准文件、报告视频对应。

## 4. 如何保证多个录制视频同步与校准

同步和校准不能只靠 GUI 自动保证，采集方法很关键。

同步建议：

- 最好使用硬件同步或同一触发源。
- 如果没有硬件同步，录制时让被试做明显的快速垂直动作，例如跳一下、快速抬手。
- 在“参数 > 同步与校准”中开启“同步交互窗口”，运行“多相机同步”阶段后检查同步图。

校准建议：

- 相机位置固定后再录制正式动作。
- 相机一旦移动，外参需要重新校准。
- 内参通常和相机镜头、焦距、分辨率有关；这些变化后应重新校准。
- 已有 Qualisys、OpenCap、EasyMocap 等校准文件时，使用 `calibration_type = convert`。
- 没有现成校准时，使用 `calibration_type = calculate`，准备棋盘格/场景点素材。
- `extrinsics_method = scene` 通常更推荐，尤其是场景中有已知三维坐标点时。

重要限制：

- 如果只运行“二维姿态识别”和“多相机同步”，可以暂时没有 `calibration/*.toml`。
- 如果要运行“人物匹配”“三维重建”“虚拟标记点增强”或“OpenSim 运动学”，必须先有相机标定结果。跳过“相机校准”只适用于项目 `calibration/` 中已经存在可用 `.toml` 标定文件的情况。
- 两台普通手机没有硬件同步并不意味着不能分析；Pose2Sim 可以估计软件同步偏移。但两台相机没有标定文件时，无法把二维关键点重建成可信三维坐标，也就无法得到可信 OpenSim 关节角。
- 常见报错 `No .toml calibration file found in ...\calibration` 表示后续步骤需要 `Calib.toml`，但项目 `calibration/` 文件夹为空或没有 `.toml` 文件。

## 5. 是否支持连接多摄像头实时录制

当前不支持。Pose2Sim 的常规输入是已经录制好的视频和校准文件/校准素材；它负责后续的二维姿态识别、多机位同步、三维重建、滤波和 OpenSim 运动学。官方文档也要求先从至少两个视角拍摄被试，并把视频放入项目 `videos/`，同步阶段用于处理非硬件同步的录制视频。

本 GUI 当前也不直接控制多台摄像头，不提供实时录制、实时硬件触发、实时校准或实时同步。推荐做法是：

- 用相机厂商软件、OBS、ffmpeg、Qualisys、Vicon、OptiTrack、Caliscope、FreeMoCap、OpenCap 等采集工具录制视频。
- 如果没有硬件同步，录制时加入明显同步动作，例如快速跳跃、快速抬手、拍手或闪光。
- 相机固定后采集校准素材或导出第三方校准文件。
- 回到本 GUI，选择项目文件夹，导入视频和校准素材，调整参数，然后运行 Pose2Sim 流程。

以后可以另做“采集模块”，但那会是独立功能：需要枚举摄像头、统一分辨率/帧率/编码、同步触发、丢帧检测、磁盘写入监控和采集日志，不应混在当前 Pose2Sim 后处理流程里草率实现。

## 6. GUI 是否覆盖 Pose2Sim 功能

已覆盖主流程：

- 项目选择与检查。
- 新建项目和标准文件夹。
- 导入录制视频。
- 新手常用参数。
- 完整 `Config.toml` 高级编辑。
- 八个 Pose2Sim 主阶段。
- 实时日志。
- `.mot` 转 Excel。
- 流程成功结束后自动把 Pose2Sim 结果同步到 `output/pose2sim_results/<项目名>/`。
- 流程成功结束后自动把 `.mot` 生成 Excel 和 HTML 交互报告。
- HTML 报告优先同步显示所有处理后视频 `pose/*_pose.mp4`；如果还没有处理后视频，则回退到项目 `videos/` 中的原始视频。报告会把视频复制/转码到报告目录的 `media/` 子文件夹，减少浏览器无法播放本地跨目录视频或非兼容编码的问题。

新手页没有暴露所有 Pose2Sim 参数，这是有意设计。未暴露但仍可在高级 TOML 中编辑的内容包括：

- 自定义 RTMlib 模型字典。
- 自定义骨架树。
- DeepLabCut、AlphaPose、OpenPose 等外部工具产生结果后的转换流程。
- 尚未完全实现或容易误用的选项，如左右交换处理、畸变点处理、移动相机、keypoints 外参自动校准。
- 非常细的滤波器参数、多人匹配参数、批处理排除列表等。

这种设计更适合初学者：常用安全参数表单化，复杂参数保留在高级 TOML。
