# Pose2Sim-GUI 中文使用说明

## 1. 软件定位

本 GUI 是 Pose2Sim 的 Windows 桌面图形界面。它面向不会写代码和 TOML 的运动分析用户：用户选择项目文件夹、导入视频、按向导准备校准素材、运行分析，GUI 自动保存配置、运行 Pose2Sim，并输出 HTML 与 Excel 报告。

本 GUI 不负责连接手机或摄像头做实时录制、硬件触发、实时同步或实时校准。它处理已经录制好的多机位视频。

## 2. 推荐项目结构

每个动作分析应使用一个独立项目文件夹：

```text
项目名/
  Config.toml
  videos/
    front.mp4
    back.mp4
  calibration/
    Calib.toml
    intrinsics/
      front/
      back/
    extrinsics/
      front/
      back/
  pose/
  pose-sync/
  pose-associated/
  pose-3d/
  kinematics/
  reports/
```

GUI 新建项目时默认建议放在：

```text
D:\Application\Biomechanics\Pose2sim\output\pose2sim_results\<项目名>\
```

这样 Pose2Sim 结果和 GUI 报告都会集中在同一个项目结果文件夹中。`input/`、`output/` 和 `projects/` 不会推送到 GitHub。

## 3. 新手向导流程

请优先使用“向导”页，而不是直接进入高级流程勾选。

1. 点击“选择/新建项目”。如果项目里没有 `Config.toml`，GUI 会自动创建。
2. 点击“导入正式动作视频”，把每台相机录到的同一段动作视频复制到 `videos/`。
3. 点击“创建校准文件夹”。GUI 会按视频名创建 `calibration/intrinsics/<相机名>/` 和 `calibration/extrinsics/<相机名>/`。
4. 按采集说明补拍或导入校准素材。
5. 在校准向导里选择校准路径，填写棋盘格参数、外参方法和 scene 场景点。
6. 点击“保存校准设置到 Config.toml”。
7. 点击“运行校准”，生成 `calibration/*.toml`。
8. 点击“只检查 2D/同步”，确认人体识别和同步质量。
9. 点击“运行完整 3D + 报告”，生成三维轨迹、OpenSim 运动学和报告。

如果没有 `calibration/*.toml`，GUI 会禁用“运行完整 3D + 报告”。这是为了避免用户生成没有几何依据的三维结果。

## 4. Redmi / 普通手机校准流程

普通手机没有硬件同步和专业标定文件时，推荐重新录制一组完整素材：

### 4.1 拍摄前固定设置

- 两台手机都固定在三脚架或稳定支架上。
- 分辨率、帧率、横竖屏保持一致，例如 1080p 60 fps。
- 尽量锁定焦距、曝光和白平衡。
- 校准和正式动作之间不要移动手机。

### 4.2 内参校准

内参描述每台手机自身镜头的几何特性。

- 每台手机单独拍棋盘格或 Charuco 板。
- 标定板要覆盖画面中心、四角、近处、远处和不同倾斜角。
- 把素材放入 `calibration/intrinsics/<相机名>/`。
- 在校准向导填写内角点数和方格边长。方格边长单位为毫米，应按实际打印或购买标定板测量。

### 4.3 外参校准

外参描述每台手机在同一个真实空间中的位置和朝向。相机一旦移动，外参必须重做。

推荐使用 `scene` 方法：

- 在场地中准备至少 10 个分散的可测量 3D 点。
- 记录每个点的 X/Y/Z 坐标，单位建议统一使用米。
- 每台手机拍摄能看到这些点的短视频或图片，放入 `calibration/extrinsics/<相机名>/`。
- 在校准向导的场景点表格中填写点名、X、Y、Z。
- 运行校准时，Pose2Sim 会提示你在每台相机画面中点击对应点。

### 4.4 正式动作和同步

没有硬件同步时，正式动作视频中必须有清晰同步事件。推荐在动作开始做一次快速垂直动作，例如跳一下、快速抬手、拍手或闪光。Pose2Sim 会用 2D 关键点运动估计多相机时间偏移。

## 5. Pose2Sim 会输出什么

常见输出如下：

- `pose/`：2D 姿态 JSON 和可能的关键点叠加视频。
- `pose-sync/`：同步后的 2D 姿态结果。
- `pose-associated/`：多视角人物匹配结果。
- `pose-3d/`：三维轨迹，通常包含 `.trc`。
- `kinematics/`：OpenSim 运动学结果，通常包含 `*_scaled.osim`、`.mot` 和日志。
- `reports/`：GUI 自动生成的 HTML 交互报告和 Excel 关节活动度表。

HTML 和 Excel 报告的数据源是 OpenSim IK 生成的 `kinematics/*.mot`，不是从原始 2D/3D 坐标重新计算。

## 6. 高级模式

“参数”页保留常用安全参数和完整 `Config.toml` 编辑；“流程/高级”页保留 Pose2Sim 八个阶段的手动勾选。高级模式适合调试和复用中间结果，但普通用户应优先使用“向导”页的三个按钮。

如果没有 `calibration/*.toml`，不要在高级模式中手动运行人物匹配、三维重建、滤波、虚拟标记点增强或 OpenSim 运动学。
